# 一.项目优化

## 1.编译阶段优化

- 开发环境时重复构建更快
  - include:缩小编译范围
  - exclude:排除相关文件夹
  - modules:配置模块的查找范围
  - alias
  - external
  - 编译缓存
  - 开启多进程
- 生产环境时文件更小，加载更快
  - 开启 tree-sharking
  - scope-hosting
  - splitChunks
  - 提供 node 的空 mocks
  - 持久化缓存
